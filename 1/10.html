1. Security Headers Obbligatori

  • ‚úì Content-Security-Policy: default-src 'self', blocca inline scripts non autorizzati
  • ‚úì Strict-Transport-Security: max-age 2 anni, includeSubDomains, preload
  • ‚úì X-Frame-Options: DENY (previene clickjacking)
  • ‚úì X-Content-Type-Options: nosniff (previene MIME sniffing)
  • ‚úì Referrer-Policy: strict-origin-when-cross-origin
  • ‚úì Permissions-Policy: Disabilita feature non usate (camera, microphone, geolocation)

üí° Implementazione: Usa Helmet.js (Node), SecurityHeaders middleware (Python/Django), Spring Security (Java)


2. HTTPS & TLS - Configurazione Sicura

  • ‚úì TLS 1.3 / 1.2 only: Disabilita TLS 1.0, 1.1 e SSLv3
  • ‚úì Strong Ciphers: ECDHE + AES-GCM, no RC4, no MD5
  • ‚úì Certificati validi: Let's Encrypt auto-renewal, wildcard se necessario
  • ‚úì HSTS Preload: Registra su hstspreload.org dopo test
  • ‚úì Redirect HTTP‚ÜíHTTPS: Automatico 301, no contenuti misti
  • ‚úì OCSP Stapling: Abilita per performance e privacy

üîç Testing: SSL Labs (ssllabs.com/ssltest), testssl.sh, Mozilla Observatory


3. Web Server Hardening

  • ‚úì Nascondi versioni: Server tokens off, no banner informativi
  • ‚úì Disable directory listing: No autoindex
  • ‚úì Limita metodi HTTP: Solo GET/POST/PUT/DELETE necessari
  • ‚úì Rate limiting: Soglie per IP su login (5/min), API (100/min)
  • ‚úì Timeout configurati: Client body/header timeout brevi (10-30s)
  • ‚úì File upload limits: Max size appropriato, whitelist MIME types
  • ‚úì Blocco bot malevoli: User-agent filtering, Captcha su form pubblici

4. WAF (Web Application Firewall)

  • ‚úì OWASP ModSecurity CRS: Rule set aggiornato per OWASP Top 10
  • ‚úì Cloud WAF: Cloudflare, AWS WAF, Azure Application Gateway
  • ‚úì Custom rules: Blocco pattern specifici per la tua app
  • ‚úì Geo-blocking: Limita accesso admin a regioni/IP specifici
  • ‚úì DDoS protection: Rate limiting avanzato, challenge pages

5. Container & Orchestration Security

  • ‚úì Base images minimal: Alpine, distroless - meno superficie attacco
  • ‚úì Non-root user: USER directive, no privileged containers
  • ‚úì Read-only filesystem: Dove possibile, tmpfs per temporary
  • ‚úì Image scanning: Trivy/Clair in CI/CD, fail on Critical
  • ‚úì Network policies: Kubernetes NetworkPolicy per isolamento
  • ‚úì Secrets management: Vault, Kubernetes Secrets (encrypted at rest)
  • ‚úì Resource limits: CPU/Memory limits per prevenire DoS

6. Infrastructure as Code Security

  • ‚úì Policy as Code: Checkov, tfsec per Terraform/CloudFormation
  • ‚úì Least privilege IAM: Ruoli specifici, no wildcard permissions
  • ‚úì Encryption at rest: Volumi EBS, S3 buckets, database encryption
  • ‚úì Security groups stretti: Solo porte necessarie, IP whitelisting
  • ‚úì Logging abilitato: CloudTrail, VPC Flow Logs, Application Logs

Best Practice: Implementa security headers completi (test con securityheaders.com). TLS 1.3 + HSTS mandatory. Rate limiting su tutti endpoint pubblici. WAF con rule set OWASP. Container security: minimal images, non-root, scanning automatico. Revisione configurazioni quarterly. Automated compliance checking (CIS benchmarks).
