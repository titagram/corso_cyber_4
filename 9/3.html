Spiegazione Passo-Passo

Questo attacco richiede un balletto tra 3 attori:

  1. L'Innesco: L'attaccante invia la stringa malevola alla Vittima (es. in un form di login).
  2. La Trappola (LDAP): La Vittima "chiama" il server LDAP dell'attaccante.
  3. Il Rinvio: Il server LDAP risponde: "Non ce l'ho io, vai a scaricarlo da quel server Web".
  4. Il Payload (HTTP): La Vittima scarica il codice Java malevolo dal server Web.
  5. L'Esecuzione: La Vittima esegue il codice. Boom.
