1.1 Lo Stack LAMP: Componenti e Confini
  • Linux: Base del sistema, gestisce utenti e permessi (ACLs).
  • Apache/Nginx: Web server che interpreta le richieste HTTP.
  • MySQL/MariaDB: Database per la persistenza dei dati.
  • PHP: Linguaggio di scripting in cui è scritto WordPress.
Approfondimento

Partiamo dalle fondamenta: lo stack LAMP. Ogni componente è un potenziale punto di attacco. Su Linux, la gestione dei permessi dei file è la prima linea di difesa. Apache e Nginx sono il volto pubblico del sito; una loro errata configurazione può esporre file sensibili.

MySQL contiene il "tesoro": utenti, post, opzioni. Infine, PHP esegue il codice di WordPress. Capire come questi 4 elementi comunicano e sono isolati tra loro è cruciale per la sicurezza.
