1.5 Hardening di Nginx/Apache
  • Bloccare l'esecuzione di script PHP in cartelle sensibili.
  • Nginx in wp-content/uploads:
location ~* /uploads/.*\.php$ { deny all; }
  • Apache con .htaccess:
php_flag engine off
Approfondimento

Una delle misure di hardening più efficaci è dire al web server di non eseguire script PHP dove non dovrebbero esserci. La cartella wp-content/uploads è il posto perfetto per applicare questa regola.

Con una semplice direttiva di configurazione in Nginx o un file .htaccess in Apache, potete bloccare l'esecuzione di PHP, rendendo innocue le web shell caricate lì. Questa è una defense in depth essenziale.
