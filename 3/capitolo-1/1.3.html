1.3 Permessi: Dev (777) vs. Prod (755/644)
  • 777: Lettura, scrittura ed esecuzione per tutti. Errore comune in dev.
  • 755 per le directory: Il proprietario può fare tutto, gli altri leggere ed eseguire.
  • 644 per i file: Il proprietario può leggere/scrivere, gli altri solo leggere.
  • Obiettivo: Il web server non deve poter scrivere sul codice sorgente.
Approfondimento

Un errore classico è usare chmod -R 777 per "risolvere" problemi di permessi. In produzione, questo è un disastro. Significa che se un attaccante trova una falla che permette la scrittura di file, può modificare qualsiasi file, inclusi i file del core di WordPress, per installare una backdoor.

La regola d'oro in produzione: le directory a 755 e i file a 644. L'utente www-data deve poter scrivere solo in directory specifiche come wp-content/uploads.
