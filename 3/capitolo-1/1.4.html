1.4 "Write Access = RCE" in PHP
  • Se un attaccante può scrivere un file .php...
  • ...può semplicemente richiederlo via browser per eseguirlo.
  • Questo è Remote Code Execution (RCE).
  • Vettore comune: vulnerabilità di file upload non protette.
Approfondimento

Questo è un concetto fondamentale in PHP. A differenza di ambienti compilati, se un attaccante può caricare o modificare un file .php nella document root del web server, ha ottenuto una RCE. Non serve altro.

Basta che carichi una web shell (un file PHP che accetta comandi via POST/GET) e può eseguire qualsiasi comando come utente www-data. Ecco perché le vulnerabilità di file upload sono così critiche in WordPress.
