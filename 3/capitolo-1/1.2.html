1.2 Il Ruolo dell'utente www-data
  • Utente www-data (o apache) esegue il processo PHP.
  • Tutti i siti e i plugin condividono lo stesso utente.
  • Una vulnerabilità in un plugin = compromissione dell'intero utente.
  • Rischio: Scrittura di file in aree non autorizzate.
Approfondimento

L'utente www-data è il cuore del problema di isolamento in molti setup PHP. Se avete più siti sullo stesso server, una falla in uno di essi permette all'attaccante di leggere e scrivere file di tutti gli altri siti eseguiti con lo stesso utente.

Non è come in un ambiente containerizzato dove ogni sito ha il suo filesystem. Se www-data può scrivere in una directory, qualsiasi script PHP eseguito sotto www-data può farlo. Questo è un punto critico per la sicurezza multi-tenant.
