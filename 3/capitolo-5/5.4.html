5.4 Template Hierarchy
  • Una volta ottenuti i risultati dalla query, WordPress deve decidere quale file del tema usare.
  • Es. per una pagina con slug "contatti", cerca:
  • 1. page-contatti.php
  • 2. page.php
  • 3. singular.php
  • 4. index.php
Approfondimento

Questo è il "routing" di WordPress. Non c'è un file di rotte, ma una convenzione sui nomi dei file. WordPress cerca il file più specifico disponibile nel tema, e se non lo trova, risale la gerarchia fino a index.php.

Dal punto di vista della sicurezza, significa che un attaccante che può scrivere file nel tema può creare un template per un URL specifico per eseguire codice solo quando quell'URL viene visitato, rendendo la backdoor più difficile da individuare.
