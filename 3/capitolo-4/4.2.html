4.2 Serializzazione e PHP Object Injection
  • WordPress salva dati complessi (array, oggetti) come stringhe serializzate nel DB.
  • Se un attaccante può controllare una stringa che viene poi deserializzata (unserialize())...
  • ...può potenzialmente scatenare una vulnerabilità di PHP Object Injection.
  • Questo può portare a RCE, a seconda dei "gadgets" (classi) disponibili.
Approfondimento

La serializzazione è un modo per trasformare un oggetto PHP in una stringa per poterlo salvare. Il problema è che la funzione unserialize() può essere molto pericolosa se usata su dati controllati dall'utente.

Un attaccante può creare una stringa serializzata malevola che, una volta deserializzata, esegue codice tramite metodi "magici" come __wakeup() o __destruct(). È una classe di vulnerabilità complessa ma molto potente, spesso usata in exploitation chains avanzate.
