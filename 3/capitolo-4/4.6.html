4.6 Privilege Escalation modificando il DB
  • Scenario: un attaccante trova una SQL Injection.
  • 1. Trova l'ID di un utente a basso privilegio.
  • 2. Aggiorna il campo wp_capabilities in wp_usermeta:
UPDATE wp_usermeta SET meta_value = 'a:1:{s:13:"administrator";b:1;}' WHERE user_id = [ID] AND meta_key = 'wp_capabilities';
  • 3. L'attaccante ora può loggarsi come admin.
Approfondimento

Questo è uno scenario di attacco da manuale. Non serve nemmeno creare un nuovo utente, che potrebbe essere notato. Basta promuoverne uno esistente.

Una singola query SQL è tutto ciò che serve per passare da una SQL Injection a un controllo completo del pannello di amministrazione di WordPress. Ecco perché prevenire le SQLi è assolutamente critico.
