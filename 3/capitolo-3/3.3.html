3.3 La cartella `uploads` e le Web Shell
  • wp-content/uploads/ è dove finiscono i media caricati.
  • Deve essere scrivibile da www-data.
  • Se un attaccante riesce a bypassare i controlli e a caricare un file .php qui...
  • ...ha una web shell persistente (se l'esecuzione di PHP non è bloccata).
Approfondimento

La cartella uploads è il campo di battaglia principale. È la porta di ingresso per le web shell. Molte vulnerabilità nei plugin riguardano la gestione insicura degli upload.

L'attaccante cerca di mascherare un file PHP come se fosse un'immagine (es. shell.php.jpg) per bypassare i controlli. Se ci riesce, e se il server è configurato per eseguire PHP in quella cartella, il gioco è fatto. Questo è il motivo per cui bloccare l'esecuzione di PHP in uploads è critico.
