3.7 Protezione tramite `.htaccess`
  • File di configurazione di Apache che agisce a livello di directory.
  • Può essere usato per:
  • • Bloccare l'accesso a wp-config.php.
  • • Impedire l'esecuzione di PHP in wp-content/uploads.
  • • Bloccare l'hotlinking di immagini.
  • ⚠️ Attenzione: se www-data può scrivere su .htaccess, può aggirare le regole.
Approfondimento

Il file .htaccess è un'arma a doppio taglio. È potente per l'hardening: potete definire regole di sicurezza granulari. Ma se l'attaccante riesce a modificarlo, può disabilitare tutte le vostre protezioni.

Può anche creare nuove regole a suo vantaggio (es. dire ad Apache di trattare i file .jpg come file .php). Quindi, anche il file .htaccess deve avere permessi restrittivi (644) e non essere scrivibile da www-data.
