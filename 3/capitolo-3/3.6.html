3.6 Bloccare l'editing da Dashboard
  • WordPress permette agli admin di modificare i file di temi e plugin dalla dashboard.
  • Se un account admin viene compromesso, l'attaccante può inserire una backdoor.
  • Aggiungere in wp-config.php:
define('DISALLOW_FILE_EDIT', true);
  • Questa costante disabilita l'editor.
Approfondimento

L'editor di file integrato è comodo in sviluppo, ma è un rischio enorme in produzione. Se un attaccante ruba la password di un admin, può semplicemente andare nell'editor, aggiungere codice malevolo a un file del tema e ottenere una RCE.

Disabilitare questa funzione con la costante DISALLOW_FILE_EDIT è una delle prime cose da fare in un hardening. Rimuove un intero vettore di attacco che viene spesso sfruttato dopo una compromissione iniziale di credenziali.
