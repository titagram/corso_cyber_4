3.5 File Nascosti Pericolosi: `.git` e `.env`
  • Se si usa git per il deploy, la cartella .git potrebbe essere esposta.
  • Un .git esposto permette di scaricare l'intero codice sorgente, inclusi file con password.
  • I file .env, usati da molti tool moderni, contengono spesso credenziali in chiaro.
  • Il web server deve bloccare l'accesso a questi file/cartelle.
Approfondimento

Un errore di deploy comune è lasciare la cartella .git accessibile via web. Esistono tool che possono ricostruire l'intero repository partendo da essa, rivelando non solo il codice attuale, ma anche la sua storia.

Questo include commit dove magari sono state inserite e poi rimosse delle password. Stesso discorso per i file .env. Sono errori che trasformano una configurazione da "sicura" a "completamente esposta" in un attimo. Sempre verificare con scanner automatici.
