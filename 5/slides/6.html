Capitolo 1: Network Discovery & Fingerprinting

Slide 1.5: Robots.txt e Sitemap Analysis
Teoria: La Mappa del Tesoro

Il file robots.txt indica ai crawler quali percorsi NON indicizzare. Ironicamente, è una lista di risorse "interessanti" per un pentester.


Cosa cercare:

  • Directory di amministrazione (/wp-admin/)
  • Backup nascosti (/backup/, /old/)
  • Aree private (/private/, /staging/)
  • File di configurazione

La sitemap.xml elenca tutte le pagine pubbliche del sito, utile per mappare la struttura.


Security Insight: "Disallow" in robots.txt NON blocca l'accesso! È solo un "suggerimento" ai crawler onesti. Le directory vietate sono spesso le più interessanti.

Lab: Esplorazione File Pubblici

Lettura robots.txt:

┌──(kali㉿kali)-[~] └─$ curl http://vulnerable-wp.local/robots.txt

Output tipico WordPress:

User-agent: * Disallow: /wp-admin/ Disallow: /wp-includes/ Disallow: /wp-content/plugins/ Disallow: /wp-content/uploads/private/ Allow: /wp-admin/admin-ajax.php Sitemap: http://vulnerable-wp.local/sitemap.xml

Lettura sitemap:

└─$ curl -s http://vulnerable-wp.local/sitemap.xml | head -20
