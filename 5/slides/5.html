Capitolo 1: Network Discovery & Fingerprinting

Slide 1.4: WAF Detection
Teoria: Il Guardiano Invisibile

Un Web Application Firewall (WAF) ispeziona il traffico HTTP e blocca richieste malevole. È fondamentale sapere se c'è un WAF prima di procedere.


WAF comuni:

  • Cloudflare - Header: cf-ray, __cfduid
  • AWS WAF - Header: x-amzn-RequestId
  • ModSecurity - Spesso silenzioso
  • Wordfence - Plugin WP specifico
  • Sucuri - Header: x-sucuri-id

Security Insight: Rilevare un WAF non significa fermarsi! Molti WAF sono mal configurati o bypassabili. Adattiamo la strategia: richieste più lente, payload offuscati, tecniche di evasione.

Lab: Rilevamento con Wafw00f

Comando base:

┌──(kali㉿kali)-[~] └─$ wafw00f http://vulnerable-wp.local

Output (Nessun WAF rilevato):

______ / \ ( W00f! ) \______/ [*] Checking http://vulnerable-wp.local [+] The site http://vulnerable-wp.local is not behind a WAF. [~] Number of requests: 8

Se ci fosse Cloudflare:

[+] The site is behind Cloudflare (Cloudflare Inc.) WAF.
