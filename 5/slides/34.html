Capitolo 5: Post-Exploitation

Slide 5.3: Data Exfiltration (PoC)
Teoria: Dimostrare l'Impatto

Per dimostrare la gravità del breach, dobbiamo mostrare quali dati un attaccante potrebbe rubare.


Dati sensibili tipici in WordPress:

  • Database utenti (email, hash password)
  • Dati clienti (e-commerce)
  • Contenuti privati/draft
  • File uploads degli utenti
  • Credenziali in wp-config.php
  • API keys, token OAuth
  • Backup del database

⚠️ In un pentest reale:

  • NON esfiltrare dati reali
  • Documentare COSA potreste prendere
  • Screenshot sanitizzati (oscurare PII)

Security Insight: L'obiettivo è dimostrare l'impatto, non rubare dati. Un PoC di 5 righe dal DB è sufficiente per provare l'accesso.

Lab: Accesso al Database

Connessione al MySQL:

# mysql -u wp_user -p'Sup3rS3cr3tP@ss!' wordpress

Query sulle tabelle sensibili:

mysql> SHOW TABLES; +------------------------+ | Tables_in_wordpress | +------------------------+ | wp_users | | wp_usermeta | | wp_posts | | wp_options | +------------------------+ mysql> SELECT user_login, user_email FROM wp_users; +------------+------------------------+ | user_login | user_email | +------------+------------------------+ | admin | admin@vulnerable.local | | editor | editor@company.com | | customer1 | john.doe@gmail.com | +------------+------------------------+ -- STOP QUI per il PoC --

Per il report: Screenshot delle prime 3-5 righe, oscurando email reali se necessario.
