Capitolo 2: WordPress Enumeration

Slide 2.3: Plugin Enumeration (Passive)
Teoria: I Plugin sono la Porta d'Ingresso

La maggior parte delle vulnerabilità WordPress NON è nel core, ma nei plugin di terze parti. Identificarli è la fase più critica.


Perché i plugin sono pericolosi:

  • Sviluppati da terze parti (qualità variabile)
  • Spesso non aggiornati
  • Accesso diretto a file system e database
  • Possono introdurre RCE, SQLi, LFI, XSS

Detection passiva: WPScan analizza il codice HTML cercando riferimenti a /wp-content/plugins/


Security Insight: Un attaccante non ha bisogno di 100 plugin vulnerabili. Ne basta UNO. Anche un plugin "innocuo" può essere il punto di ingresso se non aggiornato.

Lab: WPScan Plugin Scan Passivo
