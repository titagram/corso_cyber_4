Capitolo 1: Network Discovery & Fingerprinting

Slide 1.2: Port Scanning con Nmap
Teoria: Anatomia di una Scansione

Il port scanning rivela quali servizi sono in ascolto sul target. Ogni porta aperta è un potenziale vettore d'attacco.


Flag utilizzati:

  • -sS: SYN Scan (stealth, non completa il 3-way handshake)
  • -sC: Esegue script NSE di default
  • -sV: Rileva versione dei servizi
  • -p-: Scansiona tutte le 65535 porte
  • -T4: Timing aggressivo (più veloce)
  • -oN: Output in formato normale (file)

Security Insight: La versione esatta del software (es. Apache 2.4.38) è cruciale per cercare CVE specifiche. Un server "hardened" nasconde queste info.

Lab: Scansione Completa

Comando:

┌──(kali㉿kali)-[~] └─# nmap -sS -sC -sV -p- -T4 vulnerable-wp.local -oN scan.txt
