Capitolo 3: Vulnerability Assessment & Triage

Slide 3.2: Verifica Manuale della Versione
Teoria: Trovare la Versione Reale

Per confermare o smentire un sospetto di WPScan, dobbiamo determinare la versione esatta del plugin.


File che contengono la versione:

  • readme.txt - Campo "Stable tag"
  • readme.md - Markdown version
  • changelog.txt - Storia versioni
  • [plugin-name].php - Header PHP
  • package.json - Se usa npm

Percorso standard:

/wp-content/plugins/[nome-plugin]/readme.txt

Security Insight: L'accesso a readme.txt è una piccola "information disclosure", ma è così comune da essere considerata normale. Un admin attento potrebbe bloccarla.

Lab: Verifica con cURL

Cercare la versione nel readme:

┌──(kali㉿kali)-[~] └─$ curl -s http://vulnerable-wp.local/wp-content/plugins/simple-file-list/readme.txt | grep -i "stable tag"

Output:

Stable tag: 4.2.2

Analisi:

Versione installata: 4.2.2 Versione fix (CVE): 4.2.3 4.2.2 < 4.2.3 → VULNERABILE!

✅ Confermato: La versione 4.2.2 è vulnerabile alla RCE. Il sospetto di WPScan era corretto! Questo è un VERO POSITIVO.
