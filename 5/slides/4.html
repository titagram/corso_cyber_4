Capitolo 1: Network Discovery & Fingerprinting

Slide 1.3: Analisi degli Header HTTP
Teoria: Leggere gli Header HTTP

Ogni risposta HTTP contiene header con metadati preziosi per il pentester. Gli header più interessanti sono:


  • Server: Rivela web server e versione
  • X-Powered-By: Tecnologia backend (PHP, ASP.NET)
  • X-Frame-Options: Protezione clickjacking
  • X-XSS-Protection: Filtro XSS del browser
  • Content-Security-Policy: Policy CSP
  • Set-Cookie: Flag di sicurezza dei cookie

Security Insight: L'assenza di header di sicurezza (CSP, X-Frame-Options) indica una configurazione debole. Annotate ogni header mancante nel vostro report.

Lab: Analisi con cURL

Richiesta solo header (-I = HEAD request):

┌──(kali㉿kali)-[~] └─$ curl -I http://vulnerable-wp.local

Output:

HTTP/1.1 200 OK Date: Mon, 01 Dec 2025 10:00:00 GMT Server: Apache/2.4.38 (Debian) X-Powered-By: PHP/7.3.4 Link: <http://vulnerable-wp.local/wp-json/>; rel="https://api.w.org/" Content-Type: text/html; charset=UTF-8 ⚠️ MANCANO: X-Frame-Options, CSP, X-XSS-Protection

Verboso (-v) per vedere anche request headers:

└─$ curl -v http://vulnerable-wp.local 2>&1 | head -20
