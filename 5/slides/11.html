Capitolo 2: WordPress Enumeration

Slide 2.2: User Enumeration
Teoria: Trovare i Nomi Utente

Conoscere gli username validi è essenziale per attacchi di brute force. WordPress espone gli utenti in diversi modi:


Metodi di enumerazione:

  • REST API: /wp-json/wp/v2/users
  • Author Archives: /?author=1, /?author=2...
  • Login Error: Messaggi diversi per user valido/invalido
  • RSS Feed: Tag <dc:creator>
  • oEmbed API: /wp-json/oembed/1.0/embed

Security Insight: L'enumerazione utenti è classificata come "Informational", ma trasforma un attacco brute force da "indovina user+pass" a "indovina solo la password". Dimezza la complessità!

Lab: WPScan e REST API

Con WPScan:

┌──(kali㉿kali)-[~] └─$ wpscan --url http://vulnerable-wp.local -e u

Output:

[+] Enumerating Users (via Passive and Aggressive Methods) Brute Forcing Author IDs - Time: 00:00:01 [i] User(s) Identified: [+] admin | Found By: Author Posts - Display Name | Confirmed By: Author Id Brute Forcing [+] editor | Found By: Author Id Brute Forcing

Verifica manuale REST API:

└─$ curl -s http://vulnerable-wp.local/wp-json/wp/v2/users | jq '.[].slug' "admin" "editor"
