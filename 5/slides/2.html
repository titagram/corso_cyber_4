Capitolo 1: Network Discovery & Fingerprinting

Slide 1.1: Setup Iniziale dell'Ambiente
Teoria: Risoluzione DNS Locale

Prima di qualsiasi attività di pentesting, è fondamentale configurare l'ambiente. Vogliamo che il nostro sistema risolva il dominio vulnerable-wp.local nell'IP del target.

Il file /etc/hosts ha priorità sul DNS esterno, permettendoci di creare una mappatura locale nome→IP.


Perché è importante?

  • WordPress usa i nomi di dominio per i redirect
  • Alcuni plugin verificano il dominio
  • Simula un ambiente di produzione reale

Security Insight: Questa tecnica è usata anche negli attacchi DNS Spoofing per dirottare traffico legittimo verso server malevoli.

Lab: Configurazione /etc/hosts

Step 1: Aggiungiamo la mappatura nel file hosts:

┌──(kali㉿kali)-[~] └─# echo "10.10.10.20 vulnerable-wp.local" >> /etc/hosts

Step 2: Verifichiamo la configurazione:
