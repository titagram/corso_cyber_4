Capitolo 2: WordPress Enumeration

Slide 2.4: Plugin Enumeration (Aggressive)
Teoria: Più Rumore, Più Risultati

La modalità aggressive tenta di accedere a file comuni (readme.txt, changelog.txt) per migliaia di plugin noti.


Differenze chiave:

Passive Analizza solo HTML pubblico
Aggressive Brute force di ~100.000 plugin
Mixed Combina entrambi

Security Insight: La scansione aggressiva genera MOLTO traffico e può essere rilevata. In un pentest Red Team, evitarla. In un Vulnerability Assessment autorizzato, è indispensabile.

Lab: WPScan Aggressive Mode

Comando:

┌──(kali㉿kali)-[~] └─$ wpscan --url http://vulnerable-wp.local \ -e ap --plugins-detection aggressive

Flag:

-e ap → All Plugins --plugins-detection → aggressive/passive/mixed

Output (troncato):

[+] Enumerating All Plugins (via Aggressive Methods) Checking Known Locations - Time: 00:02:35 (98576 / 98576) 100.00% [i] Plugin(s) Identified: [+] simple-file-list | Version: 4.2.2 (80% confidence) | Found By: Readme - Stable Tag [+] wp-file-manager | Version: 6.0 | Found By: Readme - Stable Tag | [!] 1 vulnerability identified
