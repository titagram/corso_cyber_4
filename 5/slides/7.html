Capitolo 1: Network Discovery & Fingerprinting

Slide 1.6: Directory Enumeration con Gobuster
Teoria: Brute Force delle Directory

Il Directory Enumeration (o Directory Busting) consiste nel provare migliaia di nomi di directory/file comuni per scoprire risorse nascoste.


Tool principali:

  • gobuster - Veloce, scritto in Go
  • ffuf - Fuzz Faster U Fool
  • dirb - Classico, più lento
  • feroxbuster - Ricorsivo automatico

Wordlist essenziali (Kali):

  • /usr/share/wordlists/dirb/common.txt
  • /usr/share/seclists/Discovery/Web-Content/

Security Insight: Cercate file di backup (.bak, .old, .swp, ~) e config esposti (.env, wp-config.php.bak). Sono errori comuni e devastanti.

Lab: Gobuster in Azione

Comando:

┌──(kali㉿kali)-[~] └─$ gobuster dir -u http://vulnerable-wp.local \ -w /usr/share/wordlists/dirb/common.txt \ -x php,txt,bak,old -t 50

Flag importanti:

dir → Modalità directory -u → URL target -w → Wordlist -x → Estensioni da testare -t 50 → 50 thread (velocità)

Output (troncato):

/index.php (Status: 301) /wp-admin (Status: 301) /wp-content (Status: 301) /wp-includes (Status: 301) /wp-config.php.bak (Status: 200) ← JACKPOT! /xmlrpc.php (Status: 200) /readme.html (Status: 200)
