Capitolo 6: Reporting & Remediation

Slide 6.2: Scrivere un Finding Efficace
Teoria: Struttura di un Finding

Ogni vulnerabilità deve essere documentata in modo completo e riproducibile.


Elementi obbligatori:

  • Title - Chiaro e specifico
  • Severity - Critical/High/Medium/Low/Info
  • CVSS Score - Con breakdown
  • Affected Component - URL, plugin, versione
  • Description - Cos'è la vulnerabilità
  • Impact - Cosa può fare un attaccante
  • Proof of Concept - Come riprodurla
  • Remediation - Come correggerla
  • References - CVE, link documentazione

Security Insight: Il PoC deve essere riproducibile! Il cliente deve poter verificare la vulnerabilità e testare il fix.

Template: Finding Dettagliato
━━━ FINDING #1 ━━━━━━━━━━━━━━━━━━━━━━━━━ Title: Unauthenticated RCE in Simple File List Plugin Severity: CRITICAL CVSS 3.1: 9.8 (AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H) Affected: http://vulnerable-wp.local/ Plugin: Simple File List v4.2.2 Description: Il plugin non valida i file caricati, permettendo upload di script PHP arbitrari. Impact: Esecuzione di codice remoto sul server. Compromissione completa del sistema.
